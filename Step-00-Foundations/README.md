## Step 0 – Foundations of Good Software Design

### Key Learnings
- Working code is not always good code
- Good design focuses on future changes
- A class should have only one responsibility
- Responsibility means “reason to change”
- Tight coupling makes code hard to maintain and test
- Loose coupling makes systems flexible and extensible

### Important Concepts
- Responsibility
- Tight Coupling vs Loose Coupling
- Change-friendly code

### Golden Rule
One class should have one responsibility and one reason to change.
